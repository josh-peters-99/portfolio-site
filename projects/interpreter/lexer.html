The Lexer

Overview

The purpose of the lexer is to analyze the source code and convert it into data structures called "tokens" that will be passed on to the parser. For example, if the following was fed into a lexer:

let x = 3 * 5;

The lexer would output something like this: 

                    
[
    LET,
    IDENTIFIER("x")
    EQUAL_SIGN,
    INTEGER(3),
    ASTERISK,
    INTEGER(5),
    SEMICOLON
]
As you can see, the lexer breaks raw input down into small pieces or "tokens" that can then be used by the parser to create an abstract syntax tree.

Token Definition

Let's begin my defining our tokens. This is a generalization but most programming languages can be broken down into tokens that fit into 1 of 5 categories:

- Identifiers: These tokens are created by the user. They are commonly used as variable names or function names.

- Literals

- Operators

- Delimiters

- Keywords
